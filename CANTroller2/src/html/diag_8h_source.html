245// Detectable transducer-related failures :: How we can detect them
246// Brakes:
247// * Pressure sensor, chain linkage, or vehicle brakes problem :: Motor retracted with position below zeropoint, but pressure did not increase.
248// * Pressure sensor zero point miscalibration (no force on pedal) :: Minimum pressure reading since startup has never reached 0 PSI or less (cal is too high), or, is more than a given margin below 0. * Note this can also be an auto-calibration approach
249// * Pressure sensor max point miscalibration (full force on pedal) :: When target set to max pressure, after motor moves to the point position isn't changing, the pressure reading deviates from max setting by more than a given margin. * Note this can also be an auto-calibration approach
250// * Position sensor problem :: When pressure is not near max, motor is driven more than X volt-seconds without position change (of the expected polarity).
251// * Brake motor problem :: When motor is driven more than X volt-seconds without any change (of the expected polarity) to either position or pressure.
252// * Brake calibration, idle high, or speedo sensor problem :: Motor retracted to near limit, with position decreased and pressure increased as expected, but speed doesn't settle toward 0.
253// * Pressure sensor problem :: If pressure reading is out of range, or ever changes in the unexpected direction during motor movement.
254// * Position sensor or limit switch problem :: If position reading is outside the range of the motor limit switches.
255// Steering:
256// * Chain derailment or motor or limit switch problem :: Motor told to drive for beyond X volt-seconds in one direction for > Y seconds.
257// Throttle/Engine:
258// * AirVelo/MAP/tach sensor failure :: If any of these three sensor readings are out of range to the other two.
259// Tach/Speedo:
260// * Sensor read problem :: Derivative of consecutive readings (rate of change) spikes higher than it's possible for the physical rotation to change - (indicates missing pulses)
261// * Disconnected/problematic speed sensor :: ignition is on, tach is nonzero, and runmode = hold/fly/cruise, yet speed is zero. Or, throttle is at idle and brake pressure high for enough time, yet speed readings are nonzero
262// * Disconnected/problematic tach sensor :: runmode is hold/fly/cruise, ignition is on and speed increases, but tach is below idle speed
263// Temperature:
264// * Engine temperature sensor problem :: Over X min elapsed with Ignition on and tach >= low_idle, but engine temp is below nominal warmup temp.
265// * Cooling system, coolant, fan, thermostat, or coolant sensor problem :: Engine temp stays over ~204 for >= X min without coolant temp dropping due to fan.
266// * Axle, brake, etc. wheel issue or wheel sensor problem :: The hottest wheel temp is >= X degF hotter than the 2nd hottest wheel.
267// * Axle, brake, etc. wheel issue or wheel/ambient sensor problem :: A wheel temp >= X degF higher than ambient temp.
268// * Ignition problem, fire alarm, or temp sensor problem :: Ignition is off but a non-ambient temp reading increases to above ambient temp.
269// AirVelo:
270// * Air filter clogged, or carburetor problem :: Track ratio of massairflow/throttle angle whenever throttle is constant. Then, if that ratio lowers over time by X below that level, indicates restricted air.
271// Battery:
272// * Battery low :: Mulebatt readings average is below a given threshold
273// * Inadequate charging :: Mulebatt readings average has decreased over long time period
274//
275// More ideas to define better and implement:
276// * Check if the pressure response is characteristic of air being in the brake line.
277// * Axle/brake drum may be going bad (increased engine RPM needed to achieve certain speedo) (beware going up hill may look the same).
278// * E-brake has been left on (much the same symptoms as above? (beware going up hill may look the same)
279// * Carburetor not behaving (or air filter is clogged). (See above about engine deiseling - we can detect this!)
280// * After increasing braking, the actuator position changes in the opposite direction, or vise versa.
281// * Changing an actuator is not having the expected effect.
282// * A tunable value suspected to be out of tune.
283// * Check regularly for ridiculous shit. Compare our variable values against a predetermined list of conditions which shouldn't be possible or are at least very suspect. For example:
284// A) Sensor reading is out of range, or has changed faster than it ever should.
285// B) Stopping the car on entering hold/shutdown mode is taking longer than it ever should.
286// C) Mule seems to be accelerating like a Tesla.
287// D) Car is accelerating yet engine is at idle.
288// * The control system has nonsensical values in its variables.
289

291// 1. Detect brake chain is not connected (evidenced by change in brake position without expected pressure changes)
292// 2. Detect obstruction, motor failure, or inaccurate position. Evidenced by motor instructed to move but position not changing even when pressure is low.
293// 3. Detet brake hydraulics failure or inaccurate pressure. Evidenced by normal positional change not causing expected increase in pressure.
